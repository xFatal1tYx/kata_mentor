Что такое нативные методы, их плюсы и минусы?

Ключевое слово native применяется, чтобы указать, что метод реализован не в файле Java, а на другом языке программирования.

Нативные методы полезны, когда:
- Необходимо вызвать библиотеку из Java, которая написана на другом языке.
- Нужен доступ к системным или аппаратным ресурсам, к которым можно получить доступ только используя другой язык (как правило, С). 

Недостатки использования библиотек native методов:
- JNI (Java Native Interface, быстрее) / JNA (JN Access, легче) могут дестабилизировать JVM, особенно если вы попытаться сделать что-то сложное. 
Если native метод делает что-то неправильно, есть вероятность аварийного завершения JVM.
- Программу с native кодом сложнее дэбажить.
- Native код требует отдельного построения фрэймворков, что может создать проблемы с переносом на другие платформы.
