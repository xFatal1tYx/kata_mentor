Терминальные операции
Терминальные ("terminal", "eager") методы — обрабатывают элементы и завершают работу стрима, 
так что терминальный оператор в цепочке может быть только один.

Промежуточные методы не сработают, пока не будет вызван терминальный
Stream нельзя переиспользовать, если он уже был использован с терминальным методом

Терминальные методы:
forEach(System.out::println) – применяет функцию, к каждому элементу стрима;
forEachOrdered() – то же, но гарантирует сохранение порядка элементов в параллельном стриме;
findFirst()/Any() – возвращает первый / любой элемент из стрима типа Optional или Optional.empty
collect(Collectors.toList() / toMap, toSet) – представляет результат стрима в виде коллекции;
toArray() – возвращает массив из значений стрима;
count() – возвращает количество элементов в стриме;
anyMatch() / allMatch() / noneMatch() – возвращает true, если условие выполняется для хотя бы одного / всех элементов / ни одного элемента;
min() / max() – возвращает минимальный или максимальный элемент по условию компаратора;
sum() / average()  – возвращает сумму / среднее арифметическое всех чисел (для числ. стримов);
reduce((x,y) -> x*y)) – возвращает результат вычисления всех элементов между собой (Optional);
