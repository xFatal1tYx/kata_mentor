В чём отличие File от Path?
Класс File – использовался с Java1 для работы с папками и файлами. Он имел ряд недостатков, главным из которых является отсутствие обработки исключений. 
Большинство методов просто возвращают true/false.

В Java 7 решили изменить работу с файлами и каталогами и добавили классы:

- Paths (с одним статик методом get(), чтобы из строки получить объект типа Path)

- Files (куда были вынесены статические методы класса File, а также добавлены новые 
(напр. copy() – для копирования, write() / read() / readAllBytes()/ readAllLines() для чтения и записи).
Большинство методов этого класса принимает и возвращают Path.

- А также интерфейс Path. В методах Path есть все синтаксические операции, из класса File, а также добавлены новые.
Большинство его методов описывается в классах для разных файловых систем (Jrt/Windows/ZipPath), и выбрасывают исключение в случае проблем.

Path.of(путь) - создать объект
getFileName() - получить имя файла/директории
getRoot() - получение корневой директории 
getParent() - получение родительской директории
startsWith() / endsWith() - проверка, начинается/заканчивается ли путь с переданного
toAbsolutePath() / isAbsolute() - преобразование пути в абсолютный/проверка абсолютный ли путь
resolve(file) - склеивание двух Path (абсолютный + относительный)
relativize(file) - нахождение пути относительно другого пути, объединение двух путей и остаток их разницы (разница путей)
normalize() – нормализует путь (убирает символы ‘.’ и ‘..’, обозначающие текущую/родительскую директорию)

Все они возвращают объекты типа Path.
