Расскажи про pull строк и pull примитивов? 

String pool — это особое место в heap’е, куда попадают объекты типа String после их создания. Он выполняет функцию кэша строк. 
Каждый раз, когда создаётся строка, она попадает в строковый пул. 
Если же на момент создания новой строки пул уже содержит такое же значение, то вместо создания нового объекта возвращается тот, что уже лежит в пуле.

Если мы самостоятельно хотим добавить строку в пул, можно использовать метод intern():
text = text.intern(); // попытается положить текущую строку в пул строк. Если обнаружит, что такое значение уже там есть, то вернёт существующий объект.

У примитивов, а точнее их обёрток, также есть пул целых чисел в промежутке от -128 до 127.
Так, если создавать Integer в этом промежутке, то вместо того, чтобы каждый раз создавать новый объект, JVM берет их из пула. 

Несмотря на то, что ссылки на несколько объектов с одинаковыми значениями могут быть равны, при изменении одного из них, создастся новый объект, не затронув остальные.
Если мы хотим создать именно новый объект, необходимо использовать ключевое слово new;
