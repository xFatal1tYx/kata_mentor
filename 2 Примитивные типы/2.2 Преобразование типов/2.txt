При применении явных преобразований мы можем столкнуться с потерей данных. 

Так, если мы попытаемся присвоить переменной число, превышающее возможный диапазон типа, то мы просто получим неверный результат.
Пр. byte b = (byte) (127 + 1); // b = -128

Или если привести int к byte, то старшие байты в двоичной системе просто отбросятся до 8 бит, и при начальном значении, превышающем этот диапазон, мы также получим ошибочный результат.
Пр. int a = 1000; byte b = (byte) a; // b = -24

Это называется переполнением. 
Подобная операция в Java не вызовет исключение, но заметить ошибку в дальнейшем может быть очень затруднительно.

А если попытаться присвоить число с плавающей точкой целочисленному типу, то мы потеряем все числа после запятой.
Пр. int i = (int) 23.98f; // i = 23
