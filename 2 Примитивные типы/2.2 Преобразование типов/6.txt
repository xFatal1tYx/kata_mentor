Расскажи про unboxing, autoboxing в обертках.

Для каждого примитивного типа, существуют классы-обёртки, которые нужны, чтобы получить ссылочный тип на примитивные типы (превратить в объект).
boolean - Boolean / byte - Byte / ... / int - Integer / char - Character

Все классы обёртки являются неизменяемыми (immutable).
Классы-обёртки могут пригодится, например при использованиях коллекций в Java, т.к. они не работают с примитивными типами.

Процедура помещения примитива в его класс-обёртку, называется boxing:
Пр. int primitive = 0; Integer reference = Integer.valueOf(primitive); 
А обратный процесс (получение примитива из обёртки), unboxing:
Пр. int backToPrimitive = reference.intValue(); 

На самом деле нет необходимости явно использовать эти методы, начиная с JDK 5, компилятор сделает это автоматически, 
это называется autoboxing / autounboxing.
Пр. Integer a = 5; int b = a;
Пр. double a = 23.5; Double b = a;
